REQUEST FOR INFORMATION.

BACKGROUND

Finnish vessel traffic service (VTS) system gathers information from e.g. radars, Automatic Identification Systems (AIS), Closed-Circuit Television (CCTV) systems, meteorological and hydrological systems and VHF direction finders. The technical types and amount of installed sensor devices inside a device category will vary over the years, for example new radars may be installed and those may be from a different manufacturer than the existing ones. It is also possible that completely new device categories are wanted to be integrated as part of the solution. The sensor devices to be integrated may also be controllable, in which case there should be means to give the needed commands back from the VTS to the devices.

Some parts of the information are also used by other public authorities and part of the information may be provided to external 3rd party service providers. It will be individually decided what information categories can be given to the different requesters, and inside a given category necessary information filtering will also be applied.

INFORMATION REQUESTED

Please do give your answers both to the top level questions as well as for the questions targeting some more specific issues:

1) Please illustrate the way your VTS solution supports ESB based setup.

1.1) Include in the illustration also your opinion on the architectural positioning of the ESB. Which systems would you connect directly to the ESB?

1.2) Include in the illustration your views on both directions of the data communication. How the data flows from the sensor devices to the VTS desktop and how the VTS system can control the sensor devices.

1.3) Describe whether your solution includes a commercial ESB as part of it, or does it use some custom built solution fulfilling the ESB characteristics.

1.4) Do you support a configuration where your VTS solution will be configured to utilize an existing ESB. I.e. can it support the case where the client would like to use the generic ESB, already utilized for other purposes, as part of the VTS solution?:

2) Please describe the high level process accomplished when new information providers are added to the system.

2.1) As one option describe the case of adding a completely new sensor device category like for example the smart buoys.

2.2) Describe also other external integration possibilities. Does your system provide a way to integrate external services as part of the VTS desktop?:

3) Please describe the high level process accomplished when new information consumers such as public authorities or external 3rd party service providers are added to the system.

3.1) Describe also whether this can be done solely by the organization running the VTS or does it involve activities from you?

3.2) Describe the process for the case where the raw information from the information providers is distributed to new consumers as well as for the case where information processed and refined by the VTS server is distributed:

4) Please list the data formats that you support out-of-the box as input and output formats with the ESB solution you utilize.

4.1) Do you support streaming media formats? For example if there would be live video feed from a CCTV camera can it be transferred via the ESB?

4.2) Describe the data format transformations available out-of the box:

5) If any public references are available, please do list your VTS system Installations, which utilize ESB. If you cannot name the specific references, please do tell at least the count of them.
